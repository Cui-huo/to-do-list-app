## 一、Public API / 入口清单与下游调用面

### 1.1 入口点

| 入口 | 文件 | 说明 |
|------|------|------|
| `NotesApp.run()` → `build()` | `ui/app.py` | 唯一启动入口：初始化 DB、服务、ScreenManager、中文字体、Windows 标题栏 |
| `MainScreen` | `ui/main_screen.py` | 主屏，`on_enter` 触发全量刷新 |
| `SettingsScreen` | `ui/settings_screen.py` | 设置页 |
| `TagManagerScreen` | `ui/tag_manager.py` | 标签管理页 |

### 1.2 Service 层 Public API

**NoteService** (`controller/note_service.py`)：
- `create / get_by_id / update / delete / undo_delete / get_undo_info / clear_undo`
- `mark_complete / mark_incomplete`
- `pin_note / unpin_note`
- `get_incomplete / get_completed`
- `add_tag / remove_tag / get_tags / get_with_tags`
- `_get_sort_preference / set_sort_preference`（`_get_sort_preference` 以下划线开头但被 UI 层直接调用）

**TagService** (`controller/tag_service.py`)：
- `create / get_all / get_by_name / update / delete`
- `set_pinned / toggle_pinned / get_pinned / get_pinned_tags`

**SearchService** (`controller/search_service.py`)：
- `search(keyword, tag_names, year, month, week, time_type)`

**ReminderService** (`controller/reminder_service.py`)：
- `create / delete / get_by_id / get_for_note / trigger / get_pending`

**ExportService** (`controller/export_service.py`)：
- `export(format, output_path)`

**SettingsService** (`controller/settings_service.py`)：
- `set_dark_mode / get_dark_mode`

**WebDAVService** (`controller/webdav_service.py`)：
- `set_credentials / get_credentials / is_configured / sync`（全部为桩）

### 1.3 跨模块调用面（关键依赖）

```
MainScreen ──► NoteService (CRUD, 排序, 标签关联, 撤销)
           ──► TagService (get_all 取标签列表)
           ──► SearchService (search)
           ──► build_add_edit_dialog (dialogs.py)
           ──► build_search_dialog (search_dialog.py)
           ──► build_confirm_dialog (dialogs.py)
           ──► NoteCard (note_card.py)
           ──► 直接调用 note_service._get_sort_preference()

SettingsScreen ──► MainScreen (apply_template, save_current_as_template,
                   delete_template, _load_templates, _save_templates,
                   _apply_title_suffix_style, _apply_func_row_style,
                   _apply_section_header_style, refresh_list, username_color...)
              ──► 直接读 app.db_conn 执行 SQL

TagManagerScreen ──► TagService, NoteService
                ──► build_confirm_dialog

SearchContent ──► 自管理标签和下拉菜单状态

NoteService._get_sort_preference() ──► UserSettings (sort_preference)
                                  被 MainScreen.toggle_sort_preference() 直接调用
```

## 四、多套实现 / 多套规则并存

| # | 冲突 / 重复 | 详情 |
|---|------------|------|
| 1 | **Toast 三重实现** | `main_screen._toast()` / `settings_screen._toast()` / `tag_manager._toast()` — 三份完全相同的代码 |
| 2 | **_get_services 三重实现** | 三个 UI 文件各自实现了相同的 `_get_services()` |
| 3 | **`_update_theme_colors` 双重实现** | `main_screen.py` 和 `settings_screen.py` 各有自己的版本 |
| 4 | **`_load_style_dict` / `_save_style_dict` 双重实现** | `settings_screen.py` 和 `main_screen.py` 各有一套 DB 读写逻辑（`main_screen` 叫 `_load_style`/`save_style`，`settings_screen` 叫 `_load_style_dict`/`_save_style_dict`，逻辑几乎相同） |
| 5 | **Chip 颜色修复三重实现** | `note_card._apply_chip_text_style` / `dialogs._fix_chip_label_color` / `search_dialog._fix_chip_label_color` — 都与 KivyMD 1.2.0 MDChipText→MDLabel 转换后颜色丢失有关，但各自独立实现 |
| 6 | **颜色选项两套** | `main_screen.edit_username` 内硬编码 `COLOR_OPTIONS`（3 色） vs `settings_screen._get_theme_color_options`（4 色，主题感知） |
| 7 | **字体选项两套** | `main_screen.edit_username` 内硬编码 `FONT_OPTIONS`（3 种） vs `settings_screen.py` 模块顶层 `FONT_OPTIONS`（3 种，值相同但独立维护） |
| 8 | **UserSettings 写入路径分散** | 至少 7 个不同位置直接 `conn.execute("INSERT OR REPLACE INTO UserSettings ...")`，无统一入口 |
| 9 | **R26 统一返回结构不统一** | `NoteService.create()` 抛 `ValueError` / 返回 `Note`；`TagService.create()` 抛 `ValueError` / 返回 `Tag`；`ExportService.export()` 返回 `(bool, str)`；`SettingsService.set_dark_mode()` 返回 `(bool, str)`。Service 层存在两套错误处理风格：抛异常 vs 返回元组 |
| 10 | **排序 SQL 有两套** | `get_incomplete()` 在有/无置顶标签时分两路SQL：一路 `SELECT DISTINCT n.* ... JOIN`，另一路 `SELECT * FROM Note WHERE ...`。列引用别名不统一（`n.updated_at` vs `updated_at`） |
| 11 | **KivyMD 版本依赖硬编码** | 多处注释 `# KivyMD 1.2.0 内部转换...`，说明代码对特定版本行为敏感；多处用 `Clock.schedule_once` 绕过初始化时序问题（`MDSwitch.active` 崩溃、`MDChipText` 转换等） |

## 五、可疑行为清单（重构前必须原样锁住）

| # | 可疑行为 | 位置 | 风险等级 |
|---|---------|------|---------|
| 1 | **`_update_theme_colors` 设置 `top_bar.md_bg_color` 在 Light 模式用 `theme.primary_color`、Dark 模式用 `theme.bg_dark`** | `main_screen.py:_update_theme_colors`, `settings_screen.py:_update_theme_colors`, `tag_manager.py:on_enter` | **中** — 如果重构统一了主题色逻辑，标题栏颜色可能意外变化 |
| 2 | **`NoteService._undo_data` 在 `get_undo_info()` 被调用时会检查超时并自动清除** | `note_service.py:get_undo_info` | **高** — 这是个**有副作用的 getter**，调它可能清掉撤销数据 |
| 3 | **`create()` 在标签不存在时通过 `ValueError` 中的 "不存在" 字符串匹配来判断是否静默跳过** | `note_service.py:create()` L45-L48 | **高** — 依赖异常消息字符串做控制流，任何消息文案修改都会破坏此逻辑。`update()` 中有同样的模式 |
| 4 | **`_get_sort_preference()` 以下划线开头但被 UI 层直接调用** | `main_screen.py:_update_sort_label`, `toggle_sort_preference` | **低** — 约定破坏，但行为明确 |
| 5 | **`db_conn` 被 UI 层直接访问执行 SQL** | `main_screen.py`, `settings_screen.py` 多处 | **中** — 绕过了 Service 层，重构时如果改了表结构，这些位置不会报编译错误 |
| 6 | **`get_incomplete()` 在有置顶标签时用 `SELECT DISTINCT` + `LEFT JOIN`，无置顶标签时用 `SELECT *`** | `note_service.py` L148-L167 | **中** — 两条 SQL 的列引用别名不同（`n.updated_at` vs `updated_at`），虽功能等价但如果在重构时合并为一条 SQL，可能引入难以察觉的差异 |
| 7 | **`reminder_service.create()` 注释说"允许过去/未来任意时间"，但没有实际的时间校验逻辑** | `reminder_service.py:create()` | **低** — 行为是"不校验"，重构时如果加校验会改变行为 |
| 8 | **`AddEditContent` 和 `SearchContent` 各自维护独立的 `_selected_tags` 状态，都有 `set_all_tags`/`get_selected_tags`/`_make_chip`/`_toggle_tag`** | `dialogs.py`, `search_dialog.py` | **中** — 两套几乎相同的标签选择逻辑，但 `AddEditContent` 多了 `MAX_TAGS_PER_NOTE` 上限检查。合并时上限逻辑可能意外带入搜索面板 |
| 9 | **`UserSettings.username` 存储是纯文本 `row[0]`，但其他设置都是 `json.dumps/loads`** | `main_screen.py:_load_username` vs `_load_style` | **低** — 存储格式不一致，重构统一序列化可能需数据迁移 |
| 10 | **`SettingsScreen._get_theme_color_options` 在暗色模式返回"白色"作为第一个颜色，但在 `main_screen.edit_username` 中颜色选项不含白色** | `main_screen.py` (COLOR_OPTIONS 3色) vs `settings_screen.py` (_get_theme_color_options 4色) | **中** — 两套颜色列表不一致，用户可能在不同入口看到不同可选颜色 |
| 11 | **`_handle_delete` 中 Toast 写死为 "应用未关闭的12小时内..."** 但 `UNDO_TIMEOUT_SECONDS = 12 * 3600`，文案描述与实际行为一致但未引用常量 | `main_screen.py:_handle_delete` | **低** — 如果修改 `UNDO_TIMEOUT_SECONDS`，Toast 文案不会自动更新 |
| 12 | **`SettingsScreen.open_group1_dialog` 等跨主题编辑有守卫逻辑**（当前主题不符时拒绝并 Toast），但 `_on_save` 中又做了一遍 `is_dark == current_dark` 检查做不同处理 | `settings_screen.py` | **中** — 双重守卫：入口挡掉了一半，但 `_on_save` 中的另一种处理路径似乎永远执行不到（入口已挡） |
| 13 | **`_apply_title_suffix_style` 中当 `color` 为 None 时不设置 `text_color`，注释说"保留 label 当前颜色"** | `main_screen.py:_apply_title_suffix_style` | **中** — 依赖 label 在 KV 中没有显式设置 `text_color` 时的默认行为，跨主题切换后颜色是否正确不确定 |
| 14 | **`_fix_chip_label_color` 每处实现略有不同**：`dialogs.py` 版用 `is_active` 参数决定颜色；`search_dialog.py` 版相同；`note_card.py` 版用参数传递的 `rgba` 颜色。三者在 `walk()` 遍历中都找 `isinstance(w, Label)`，但没有处理可能的多层嵌套。 | 三处 | **低** — 当前能工作，但重构时容易遗漏某个变体 |

## 一、Public API / 入口清单与下游调用面

### 1.1 入口点

| 入口 | 文件 | 说明 |
|------|------|------|
| `NotesApp.run()` → `build()` | `ui/app.py` | 唯一启动入口：初始化 DB、服务、ScreenManager、中文字体、Windows 标题栏 |
| `MainScreen` | `ui/main_screen.py` | 主屏，`on_enter` 触发全量刷新 |
| `SettingsScreen` | `ui/settings_screen.py` | 设置页 |
| `TagManagerScreen` | `ui/tag_manager.py` | 标签管理页 |

### 1.2 Service 层 Public API

**NoteService** (`controller/note_service.py`)：
- `create / get_by_id / update / delete / undo_delete / get_undo_info / clear_undo`
- `mark_complete / mark_incomplete`
- `pin_note / unpin_note` — `pin_note()` 同时刷新 `updated_at`
- `get_incomplete / get_completed` — 排序三层：手动置顶(updated_at DESC) → 标签置顶(updated_at DESC) → 非置顶(sort_preference DESC)
- `add_tag / remove_tag / get_tags / get_with_tags / get_tags_batch`
- `_get_sort_preference / set_sort_preference`（`_get_sort_preference` 以下划线开头但被 UI 层直接调用）

**TagService** (`controller/tag_service.py`)：
- `create / get_all / get_by_name / update / delete`
- `set_pinned / toggle_pinned / get_pinned / get_pinned_tags`

**SearchService** (`controller/search_service.py`)：
- `search(keyword, tag_names, year, month, week, time_type)`

**ReminderService** (`controller/reminder_service.py`)：
- `create / delete / get_by_id / get_for_note / trigger / get_pending`

**ExportService** (`controller/export_service.py`)：
- `export(format, output_path)`

**SettingsService** (`controller/settings_service.py`)：
- `set_dark_mode / get_dark_mode`

**WebDAVService** (`controller/webdav_service.py`)：
- `set_credentials / get_credentials / is_configured / sync`（全部为桩）

### 1.3 跨模块调用面（关键依赖）

```
MainScreen ──► NoteService (CRUD, 排序, 标签关联, 撤销)
           ──► TagService (get_all 取标签列表)
           ──► SearchService (search)
           ──► build_add_edit_dialog (dialogs.py)
           ──► build_search_dialog (search_dialog.py)
           ──► build_confirm_dialog (dialogs.py)
           ──► NoteCard (note_card.py)
           ──► 直接调用 note_service._get_sort_preference()

SettingsScreen ──► MainScreen (apply_template, save_current_as_template,
                   delete_template, _load_templates, _save_templates,
                   _apply_title_suffix_style, _apply_func_row_style,
                   _apply_section_header_style, refresh_list, username_color...)
              ──► 直接读 app.db_conn 执行 SQL

TagManagerScreen ──► TagService, NoteService
                ──► build_confirm_dialog

SearchContent ──► 自管理标签和下拉菜单状态

NoteService._get_sort_preference() ──► UserSettings (sort_preference)
                                  被 MainScreen.toggle_sort_preference() 直接调用
```

## 四、多套实现 / 多套规则并存

| # | 冲突 / 重复 | 详情 |
|---|------------|------|
| 1 | **Toast 三重实现** | `main_screen._toast()` / `settings_screen._toast()` / `tag_manager._toast()` — 三份完全相同的代码（`MDSnackbar(MDLabel(...))`，2秒，居中），独立维护 |
| 2 | **_get_services 三重实现** | 三个 UI 文件各自实现了相同的 `_get_services()`（返回 `(note_service, tag_service, search_service)`） |
| 3 | **`_update_theme_colors` 双重实现** | `main_screen.py` 和 `settings_screen.py` 各有自己的版本。三页主题更新实际三处：`main_screen._update_theme_colors` + `settings_screen._update_theme_colors` + `tag_manager.on_enter` 内联 |
| 4 | **`_load_style` / `_save_style` 双重实现** | `settings_screen.py`（`_load_style_dict`/`_save_style_dict`）和 `main_screen.py`（`_load_style`/`save_style`）各有一套 DB 读写逻辑，功能几乎相同 |
| 5 | **Chip 颜色修复三重实现** | `note_card._apply_chip_text_style`（额外设 font_size/font_name/bold）/ `dialogs._fix_chip_label_color`（用 `is_active` 参数）/ `search_dialog._fix_chip_label_color`（同 dialogs 版） — 都与 KivyMD 1.2.0 MDChipText→MDLabel 转换后颜色丢失有关 |
| 6 | **颜色选项两套** | `main_screen.edit_username` 内硬编码 `COLOR_OPTIONS`（金/天蓝/珊瑚橙，3色） vs `settings_screen._get_theme_color_options`（白天：黑/金/天蓝/珊瑚橙；黑夜：白/金/天蓝/珊瑚橙，4色，主题感知） |
| 7 | **字体选项两套** | `main_screen.edit_username` 内硬编码 `FONT_OPTIONS`（默认/东方大楷/乐米波波，3种） vs `settings_screen.py` 模块顶层 `FONT_OPTIONS`（值相同但独立维护）+ `FONT_SIZE_OPTIONS` |
| 8 | **UserSettings 写入路径分散** | 至少 7 个不同位置直接 `conn.execute("INSERT OR REPLACE INTO UserSettings ...")`，无统一入口，无迁移机制 |
| 9 | **R26 统一返回结构不统一** | `NoteService` 抛 `ValueError`；`TagService` 抛 `ValueError`；`ExportService.export()` 返回 `(bool, str)`；`SettingsService` 返回 `(bool, str)`。Service 层存在两套错误处理风格：抛异常 vs 返回元组 |
| 10 | **排序 SQL 双分支** | `get_incomplete()` 在有/无置顶标签时分两路 SQL：一路 `SELECT DISTINCT n.* ... LEFT JOIN`（列引用 `n.updated_at`），另一路 `SELECT * FROM Note WHERE ...`（列引用 `updated_at`）。列别名不统一 |
| 11 | **KivyMD 1.2.0 版本耦合** | 多处注释 `# KivyMD 1.2.0 内部转换...`，代码对特定版本行为敏感；多处用 `Clock.schedule_once` 绕过初始化时序问题（`MDSwitch.active` 崩溃、`MDChipText` 转换、`self.ids` 生命周期等） |
| 12 | **双主题文字样式 key 隔离** | 白天模式 key 无前缀（`username_style`），黑夜模式 key 加 `dark_` 前缀（`dark_username_style`）。`_get_theme_prefix()` 集中管理前缀逻辑，但默认值字典（`LIGHT_STYLE_DEFAULTS` / `DARK_STYLE_DEFAULTS`）独立维护，共 10 个 key |

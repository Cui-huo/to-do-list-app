# 可复用操作流程 (Skills)

> 本文档沉淀开发过程中可复用的功能操作流程。
> 与 `spec.md`（功能规格）、`rules.md`（设计规则）互补。

---

## SKL-01. 便签新增流程

```
1. 用户点击 FAB (+)
2. 弹出 MDDialog：
   ├─ title 输入框（可选，≤50 字符）
   ├─ content 输入框（必填，1–5000 字符，Markdown）
   └─ 标签选择区：显示已有标签芯片，点击多选
3. 用户点击 [保存]
4. 校验：content 非空 → 通过则创建
5. 便签出现在未完成区顶部（按排序规则）
```

---

## SKL-02. 便签删除 + 撤销流程

```
1. 用户点击便签卡片删除按钮
2. 弹出确认框："确认删除？可撤销恢复（12小时内）"
3. 用户确认 →
   a. 内存暂存 {title, content, tag_names}
   b. 物理删除 Note + 级联 NoteTag/Reminder/FTS5
   c. 卡片消失
   d. 底部弹出 UndoBar：「已删除。可在 12 小时内恢复  [撤销]」
4. 分支：
   ├─ 用户点击 [撤销] → 创建新便签（随机标题 + 原内容 + 原标签）→ UndoBar 消失
   ├─ 超过 12h → 暂存清空，UndoBar 消失
   └─ 应用关闭 → 暂存清空，下次启动无 UndoBar
5. 再次删除（覆盖暂存）：上次暂存数据被覆盖，仅保留最新一条
```

## SKL-03. 标签管理流程

```
1. 入口：主屏标签按钮
2. 标签管理页：
   右上角：[+新建标签] [批量删除]
   列表每行：
     [标签名] ← 点击弹出改名对话框
     [📌置顶]  ← 点击切换置顶状态

3. 新建标签：
   点击 [+新建标签] → 输入框 + 确认 → 创建（校验：1–10 字符，不重名）

4. 改名标签：
   点击标签名 → 弹框输入新名 → 确认（不重名）

5. 置顶标签：
   点击 📌 → 置顶该标签
   超过 3 个 → 自动淘汰最早置顶的标签

6. 批量删除：
   点击 [批量删除] → 多选模式 → 勾选 → [确认删除] → 确认框 → 级联删除
```

## SKL-04. 搜索筛选流程

```
1. 展开搜索面板
2. UI 输入：
   ├─ keyword:  文本框输入（可选）
   ├─ tag_names: 点击标签芯片多选（可选）
   ├─ year/month: 下拉选择（可选）
   ├─ week: 下拉选择 1–5（可选）
   └─ time_type: 芯片切换「完成时间 | 创建时间」
3. 点击 [搜索] 触发
4. 结果以列表展示
5. 所有条件 AND 组合
```

## SKL-05. 置顶排序层级

```
未完成便签显示顺序（从上到下）：
┌──────────────────────────────┐
│ 手动置顶便签（is_pinned=1）   │ ← 按 pinned_at DESC
│   ├─ position > 0: 按 position DESC
│   └─ 其余: 按时间偏好排序
├──────────────────────────────┤
│ 标签置顶便签（含 pinned_tags）│ ← 按 updated_at DESC
│   ├─ position > 0: 按 position DESC
│   └─ 其余: 按时间偏好排序
├──────────────────────────────┤
│ 普通便签                     │ ← 按时间偏好排序
└──────────────────────────────┘

已完成便签显示顺序：
┌──────────────────────────────┐
│ 手动置顶便签（is_pinned=1）   │ ← 按 pinned_at DESC
│   ├─ position > 0: 按 position DESC
│   └─ 其余: 按 completed_at DESC
├──────────────────────────────┤
│ 普通已完CD成便签              │ ← 按 completed_at DESC
└──────────────────────────────┘
```

## SKL-06. 确认对话框模式

```
所有不可逆操作统一使用确认框：

删除便签：
  title: "确认删除"
  text:  "删除后可撤销恢复（12小时内）"
  buttons: [取消] [确认删除]

删除标签：
  title: "确认删除"
  text:  "确认删除标签「{name}」？将移除所有便签的此标签。"
  buttons: [取消] [确认删除]

批量删除标签：
  title: "确认删除"
  text:  "确认删除 {N} 个标签？将移除所有便签的这些标签。"
  buttons: [取消] [确认删除]
```

## SKL-07. 便签手动置顶流程

```
1. 用户长按便签卡片
2. 弹出上下文菜单
3. 分支：
   ├─ 便签未被置顶 → 显示 [置顶] → 点击 → is_pinned=1, pinned_at=now
   ├─ 已手动置顶   → 显示 [取消置顶] → 点击 → is_pinned=0, pinned_at=NULL
   └─ 已被标签置顶 → 显示 [置顶] 置灰 或 点击后提示"该便签已置顶"
4. 即时反馈：卡片移动到置顶区 / 移回原位
```
